**DSF 2.0.0 - First Release Candidate (RC1) Now Available!** 

We are happy to announce the DSF 2.0.0 Release Candidate (RC1). This marks a significant milestone in the development of DSF 2.0. The release of the RC1 is now publicly available on the [GitHub](https://github.com/datasharingframework/dsf/releases/tag/v2.0.0-RC1). Please note, this is a Release Candidate, do not use it in production environments. 

**Key Features in DSF 2.0.0-RC1:**

* **Workflow Engine Migration:** The DSF BPE workflow engine has been migrated from Camunda to the open-source BPMN engine Operaton.
* **New Process Plugin API v2:** The updated API offers cleaner abstraction, more capabilities and expanded metadata and more!
* **FHIR Profile Updates:** Default FHIR profiles for all supported resources have been created for the 2.0.0 version. The authorization system has been improved as well.
* **Codebase Upgrade:** The Codebase has been upgraded to Java 25, along with the latest versions of Jetty and HAPI.
* **Simplified Configuration and more unified logging Controls.**
* **Improved User Interface and User Experience.**

  
For more details, including the complete release notes, visit the official release page:
